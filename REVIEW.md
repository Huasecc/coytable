## CoyTable Review Test Instructions

### Test Steps

**1. Basic Functionality**
1. Open Figma and run CoyTable plugin
2. Verify UI displays correctly: preview area on the left, config panel on the right, toolbar on top
3. Set "Rows" to 3 and "Columns" to 3, observe live preview update
4. Set "Width" to 600px and "Height" to 200px, observe table resize

**2. Cell Editing**
1. Double-click any cell to enter edit mode
2. Press Enter to confirm
3. Press Tab to navigate to the next cell
4. Press Esc to cancel editing

**3. Row / Column Operations**
1. Drag column handles (⠿) to reorder columns
2. Drag row handles (⠿) to reorder rows
3. Hover over a row handle and click "Insert Above"
4. Hover over a column handle and click "Insert Left"
5. Try changing column type (e.g., switch to "Progress" or "Tag")

**4. Style Configuration**
1. Try different color presets in the "Preset Styles" panel
2. Adjust corner radius, font size, border colors
3. Enable zebra striping and set alternate row color
4. Toggle light/dark theme

**5. Data Templates**
1. Click "User Info" template in the "Preset Data" panel
2. Try the "Product List" template
3. Verify data is populated correctly

**6. Generate Table**
1. Click "Generate" to send the table to the Figma canvas
2. Verify the table appears correctly on canvas
3. Select the generated table — a "Re-edit" bar should appear at the bottom
4. Click "Re-edit" — data should reload into the plugin

**7. Language Switch**
1. Click "EN" in the top-right corner to switch to English
2. Click "中" to switch back to Chinese

### Known Limitations
- Requires network access to load Lucide icon library (cdn.jsdelivr.net)
- Help modal auto-opens on first launch
